# System Architecture & Development Learnings

## Phase 2: Backend API Normalization Pipeline (Completed Mar 2026)

During the integration of independent brokerage services into a single unified JSON schema, several high-level architectural patterns were strictly enforced to maintain stability and prevent repeating authentication cycles:

### 1. Unified Broker Service Aggregation
Rather than configuring specific endpoints for different geographic markets, **dynamic instantiation loops** act as the most robust defense. (e.g., Moomoo OpenD contexts natively reject requests outside explicit `TrdMarket` filters. Iterating across *all permutations* guarantees data capture regardless of the end-user's hidden environment variables).

### 2. Defeating Provider Security Restrictions Natively
Third-party APIs (specifically `questrade-api` utilizing `urllib`) frequently succumb to Web Application Firewalls (WAF) blocking requests without `Mozilla/5.0` User-Agent payloads.
- *Architectural Standard*: Monkey-patch protocol request engines seamlessly within the host process at the global scope level rather than locally rewriting massive third-party library internals.

### 3. Asynchronous Cache Persistence
Caching must prioritize disk or active states (`~/.questrade.json` and in-memory TTL caching) natively to survive Fast-API hot reloads (`uvicorn`). When swapping configuration vectors (like `.env` refresh tokens), the application loader must **actively burst the memory overlay** (`load_dotenv(override=True)`) to ensure the child processes aren't stubbornly carrying stale payloads.

### 4. Zero-Trust API Testing (OAuth Tokens)
Never run test verification scripts utilizing active production tokens dynamically generated by the user if the authorization structure employs a `grant_type=refresh_token` **Single-Use Payload**. Doing so consumes the sequence and locks out the main application. Mimic the output via independent test tokens or purely trace execution paths natively instead.

## Phase 5: GUI Rendering & Client Stabilization (Completed)

### 5. Type-Strict Native API Data Parsing
When dynamically aggregating unstructured or potentially anomalous streams (such as Moomoo rendering `"N/A"` for TSX-native asset currencies like `CA.MDA`), the Frontend validation layers must strictly fall back.
- *Architectural Standard*: Never unconditionally ingest backend metadata structures directly into strict parsers like `Intl.NumberFormat()`. A `RangeError: Invalid Currency Code` will cause massive DOM exceptions entirely breaking the React hydrate lifecycle. Always implement safe conditionals ensuring strings match standard length configurations (`cur.length === 3`) before format initialization natively safely.

## Phase 6: Auth Resilience & Structured Error Propagation (Completed Mar 2026)

### 6. Never Trust a Truthy Dict as a Successful API Response
In Python, any non-empty dict is truthy — including error payloads like `{'code': 1017, 'message': 'Access token is invalid'}`. A bare `if q.accounts:` guard silently accepts error dicts as valid, producing 0 positions with no diagnostic output.
- *Architectural Standard*: Always assert response *structure*, not truthiness. Use a dedicated `_is_token_valid()` probe that checks `isinstance(resp, dict) and 'accounts' in resp` (or the domain-specific success key). This isolates the validity logic and ensures it is reused consistently across all auth code paths.

### 7. Broker Auth Failures Must Propagate as Typed Exceptions — Never Return []
Silently returning `[]` on authentication failure masks the root cause. The UI renders "no positions" with no guidance, making the problem appear to be a data issue rather than an auth issue.
- *Architectural Standard*:
  1. On first auth failure, detect token-expiry signals (`code 1017`, HTTP `401`, string `'Unauthorized'`) and distinguish them from general errors.
  2. Attempt one automatic re-auth by busting the stale cache file and bootstrapping from the `.env` refresh token.
  3. If re-auth also fails, raise a typed `BrokerAuthError` (never `return []`). This exception carries a human-readable, actionable message.
  4. Catch `BrokerAuthError` at the FastAPI endpoint and return `{"status": "auth_error", "broker": "...", "message": "..."}` — a structured payload the frontend maps directly to a UI error card with step-by-step remediation instructions and a **Retry Connection** button.

## Phase 7: Frontend Aggregation & Currency Display (Completed Mar 2026)

### 8. Intl.NumberFormat Emits Currency-Prefixed Symbols for Non-USD Currencies
`new Intl.NumberFormat("en-US", { style: "currency", currency: "CAD" })` renders `CA$18.40`, not `$18.40`. If you also render an explicit ISO badge (" CAD") after the value, CAD rows display as `CA$18.40 CAD` — a redundant double-indicator. USD rows are unaffected because the USD symbol is just `$`.
- *Architectural Standard*: Always use `currencyDisplay: "narrowSymbol"` when rendering monetary values alongside a separate ISO badge. This forces all currencies to use the narrow `$` symbol, leaving the badge as the sole and unambiguous currency indicator: `$18.40 CAD` vs `$104.00 USD`.

### 9. Client-Side Symbol Grouping: Qty-Weighted Average is the Correct Aggregation for Avg Entry
When collapsing multiple positions for the same ticker into one row, a simple average of `average_buying_price` is financially incorrect because it ignores lot sizes. E.g. 13 shares @ $388 + 97 shares @ $323 → simple avg = $355.50, but the true cost-basis weighted avg = $328.48.
- *Architectural Standard*: Carry a running `_wtotal = qty × price` accumulator alongside the grouped row. On each merge: `new_avg = (prev_wtotal + qty × price) / new_total_qty`. This is O(n) and avoids needing to re-scan all constituent rows.

### 10. Grouping Logic Belongs at Module Scope, Not Inside useMemo Callbacks
The `groupBySymbol()` function is pure (no side effects, no hooks). Defining it at module scope (outside any component) makes it tree-shakeable and avoids recreating the function reference on every render. The `useMemo` wrapper *calling* it is sufficient to memoize the output — the function itself does not need to live inside the component.
